Anatomy of Error Handling

A `try...except` block helps catch errors. Click each highlighted part of the code below to see its role.

try: result = 10 / 0 # Potential error here except ZeroDivisionError: print("Cannot divide by zero!")

Identify all 7 parts!

The `try` keyword: Starts a block where errors might happen. Colon `:` after `try`: Marks the beginning of the `try` code suite. Risky Code: The operation(s) that could potentially raise an error. The `except` keyword: Starts a block to handle a specific error. Exception Type (`ZeroDivisionError`): Specifies which error this block catches. Colon `:` after `except`: Marks the beginning of the error handling code. Handler Code: What to do if the specified error occurs.
