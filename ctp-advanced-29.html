Refining `__init__`: Default Values

Sometimes, attributes should have a starting value if none is provided. Click the parts of this `__init__` method to see how default parameters work.

CT Links: Abstraction (hiding complexity with defaults), Decomposition (analyzing parameter variations).

def __init__(self, name, species="Unknown"):
    self.name = name
    self.species = species  # Uses provided or default value

Identify all 6 parts!

`species`: Another parameter for the constructor. `=`: The assignment operator used to set a default value. `"Unknown"`: The default string value used if `species` is not provided when creating an object. `self.name = name`: Standard attribute assignment for the required `name` parameter. `self.species = species`: Assigns the `species` attribute. It gets the value passed in, OR the default `"Unknown"` if nothing was passed for `species`. The parts like `def`, `__init__`, `self`, `name`, `()`, `:` function the same as in the basic `__init__`.
